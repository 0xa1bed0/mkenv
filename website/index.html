WHAT IS MKENV?

mkenv is a CLI tool that creates disposable Docker dev containers with zero configuration.

How it works:

  1. Run mkenv . in any project folder
  2. mkenv detects your languages, tools, and dependencies automatically
  3. It builds a hardened container with everything you need
  4. Connect your editor/AI tools to the mkenv container - all commands, builds, and installs run inside

Result: Your host machine stays clean. Your SSH keys, tokens, and credentials stay safe.

CORE FEATURES

Automatic Environment Detection

mkenv analyzes your repo and automatically installs the right languages, versions, CLIs, and build tools.

No Dockerfiles. No devcontainer.json. No configuration files.

Safe Terminal for AI Tools

Use mkenv as a sandboxed terminal for Claude Code, Cursor, VS Code, and other AI coding tools.

AI-generated commands run inside the container, not on your host machine with access to your credentials.

Disposable Containers

Each session gets a fresh container. Dependencies are cached for speed.

Something feels wrong? Delete the container and start over in seconds.

Clean Host Machine

Keep only Docker and mkenv on your laptop.

All language runtimes, packages, and tools run inside containers.

HOW IT WORKS

  1. Run mkenv . in your project folder
  2. mkenv analyzes your repo and generates a Dockerfile automatically
  3. It builds a hardened container and drops you inside
  4. All network activity and package installs are logged
  5. No sudo access - use mkenv sandbox for system changes
  6. Exit when done - the container is destroyed

Optional: .mkenv Configuration Files

Use .mkenv files to set defaults without repeating command flags:

{
  "enabled_bricks": ["claude-code", "nvim"],
  "volumes": ["~/data:/data"]
}

Place in your project root or any parent directory. Great for team-wide or organization-wide defaults. Learn more →

WHO NEEDS THIS?

  • Developers running npm install, pip install, or cargo build regularly
  • Teams working on client projects or unfamiliar codebases
  • Anyone using AI coding assistants (Claude Code, Cursor, Copilot)
  • Security-conscious developers who want isolation without complexity

SECURITY FEATURES

  • Runs as non-root user inside the container
  • Only your project folder is accessible - no access to SSH keys, tokens, or credentials
  • All network requests and package installs are logged
  • Container state is temporary - nothing persists after you exit
  • System changes require explicit approval via mkenv sandbox

If something malicious runs, delete the container and start fresh.

INSTALLATION

Current platform: macOS only (Linux and Windows/WSL coming soon)

Prerequisites: Docker Desktop or Docker Engine must be installed and running

Install via Homebrew

brew tap 0xa1bed0/mkenv
brew install mkenv

Usage

cd /path/to/your/project
mkenv .

mkenv will analyze your project, build a container, and drop you inside. Subsequent runs reuse cached dependencies for faster startup.
